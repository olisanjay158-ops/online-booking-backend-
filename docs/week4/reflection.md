# Week 4 Reflection

During Week 4 of the Capstone project, the focus was on improving the quality and reliability of the backend system rather than adding new features. A mock dataset was created to simulate realistic user input, including both valid and invalid records. Server-side validation was implemented to ensure that incorrect data such as invalid email formats, empty names, or unrealistic ages are properly rejected by the API. Structured error responses with HTTP 400 status codes were used to maintain consistent error handling. Automated negative tests were developed using pytest to verify that invalid inputs are correctly detected by the validation logic. These tests iterate through the mock dataset and confirm that the API returns appropriate error responses. This process helped strengthen understanding of validation, automated testing, and maintaining software quality in production systems.
